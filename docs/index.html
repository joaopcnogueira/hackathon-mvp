Plataforma AutoML

02 a 06 de Fevereiro

O Problema

80%

do tempo de cientistas de dados é gasto em
tarefas repetitivas

Limpeza de dados • Tratamento de nulos • Seleção de variáveis • Experimentação

Fonte: Forbes/CrowdFlower Data Science Survey, 2016

A Solução: AutoML

Automated Machine Learning

Democratizar o acesso à inteligência artificial,
permitindo que qualquer pessoa treine modelos
sem escrever código

O Desafio

Construir uma plataforma AutoML no-code

(point & click)

Que permita a qualquer usuário — técnico ou não —
treinar modelos de machine learning de forma
intuitiva e profissional

O que a plataforma deve fazer?

  1. Upload e Análise de Dados
  2. Pré-processamento Automático
  3. Treinamento de Modelos
  4. Avaliação e Comparação
  5. Deploy do Modelo

1. Upload e Análise de Dados

  • Aceitar arquivos CSV e Excel
  • Detectar automaticamente tipos de colunas
  • Identificar valores nulos e problemas
  • Exibir estatísticas descritivas

2. Pré-processamento Automático

  • Tratamento de valores faltantes (imputação)
  • Codificação de variáveis categóricas
  • Normalização/padronização de features
  • Divisão treino/teste correta
Evitar data leakage: split deve ocorrer
antes do pré-processamento

3. Tipos de Problema

Tipo Descrição Exemplo
Classificação Prever categorias Cliente vai comprar?
Regressão Prever valores contínuos Qual será o custo?
Séries Temporais Prever valores futuros Demanda do próximo mês?

4. Avaliação e Comparação

  • Calcular métricas apropriadas para cada tipo
  • Comparar performance entre algoritmos
  • Exibir ranking dos melhores modelos

5. Deploy do Modelo

API REST

Predições em
tempo real

Batch

Processar múltiplos
registros de uma vez

+ Download de artefatos (modelo treinado)

Datasets

Todos os times usam os mesmos datasets

Tema Dataset Objetivo
Classificação Online Shoppers Prever se visitante comprará
Regressão Insurance Prever custo de seguro
Séries Temporais Daily Temperature Prever valores futuros
Durante a avaliação, haverá datasets ocultos
para testar a robustez da solução

Critérios de Avaliação

Funcionalidade

10 itens por tema (classificação, regressão, séries temporais)

  • Upload funciona corretamente
  • Detecção automática do tipo de problema
  • Pré-processamento executa sem erros
  • Treinamento completa com sucesso
  • Métricas exibidas corretamente
  • Comparativo entre modelos
  • Deploy via API funciona
  • Deploy via Batch funciona
  • Interface intuitiva e responsiva
  • Sem erros ou travamentos

Critérios de Avaliação

Performance

Critério de desempate

Tipo Métrica Critério
Classificação F1-Score Maior é melhor
Regressão R² Score Maior é melhor
Séries Temporais MAPE Menor é melhor

Nota Final = (Classificação + Regressão + Séries) / 3

Dicas para os Times

  1. Foco na experiência do usuário
    A plataforma é para quem NÃO sabe programar
  2. Evitem data leakage
    Split treino/teste ANTES do pré-processamento
  3. Métricas corretas para cada problema
    Não usem métricas de classificação em regressão
  4. Testem com os datasets fornecidos
    Mas lembrem: haverá datasets surpresa
  5. Priorizem funcionalidade básica
    Melhor simples que funciona do que complexo quebrado
